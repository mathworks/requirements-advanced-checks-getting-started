MODEL CHECKS

 

Before something gets flagged at an internal design review in front of your boss and peers, lets use Simulink Check to look for issues in the design.

 

Lets start by opening Model Advisor by doing the following:

Analysis->Model Advisor-> Model Advisor

 

 

 

A "check" in Model Advisor is a rule that the tool will use to examine your model and flag anything that doesn't agree with the pre-defined criteria. The set of checks provided by Simulink Check can be seen under the By Product->Simulink Check folder. In addition to the modeling standard checks listed above, Simulink Check provides real-time Model Advisor checks (Edit Time Checking) and the ability to create custom checks.

 

 

Using Edit Time Checking

Model Advisor has a comprehensive set of checks that can be used to verify that the design complies with standards, which any Simulink user can benefit from through the Model Advisor GUI. What if you wanted to have your design checked in real time within the Simulink Editor? Edit Time Checking, a feature within Simulink Check, provides this experience. To enable Edit-Time Checking, do the following:

 

  Open the CruiseControl_MdlAdv_ReqLink.slx Simulink model file

  Go to Analysis->Model Advisor->Model Advisor->Display Advisor Checks in Editor

 

 

 

After edit-time checking has been enabled, the checks have been automatically run and the results displayed on the model canvas.

Note the name violation found for the subsystem "Compute target speed".

 

 

Open the state chart to find another violation regarding the "ON" state.

 

 

 

Adding Customized Checks to Model Advisor

Farnswerth Automotive has its own modeling standards. Youll need to be sure your design is compliant with these standards before it passes the design review stage. Simulink Check allows you to add your own Custom Checks to streamline this process. Custom checks can be added through the Model Advisor API and the sl_customization.m file. In this example, we will add three Custom Checks that are based on the Farnswerth Automotive style guide. These are defined in sl_customization.m . This sl_customization.m file can be used for various purposes including but not limited to V&V. For instance, it can be used to change block fonts, block colors and Simulink windows screen colors among other things. You can view the m file by clicking here.

 

To view the custom check options,

Do the following:

1.    Close all model files and the Model Advisor window

2.    Setup the sl_customization.m file by making sure that the sl_customization.m is present in the working directory

3.    At the MATLAB command prompt, enter the following command: >> Advisor.Manager.refresh_customizations

4.    Open the CruiseControl_MdlAdv_ReqLink.slx

 

OR

 

Click here and run the corresponding m-file that opens to do all the above steps automatically.

 

Now, Re-open Model Advisor and you should see the following window appear. If you do, click Remove existing report and continue. This just means that there is an old result in the system from your previous run that needs to be removed before continuing.

 

 

If you do not see the above window, it just means there was no result in the system and you continue with the below steps directly.

You will notice a folder under By Product named Custom Checks (see picture below).

The folder contains the following 3 customized checks:

  Check Simulink block font

  Check Simulink window screen color

  Check model optimization settings

 

These checks were scripted in MATLAB using the Simulink API and added to the Model Advisor UI using the sl_customization.m file. When sl_customization.m changes, the UI must be refreshed with the sl_refresh_customizations command.

 

Let's run through all three of these custom checks:

1.    Select and check Custom Checks.

2.    Click on the Run Selected Checks button (see picture below).

 

 

 

 

3. After running the checks, select the first one - Check Simulink block font - which has failed.

4. Click on the Fix block fonts button (see picture below).

 

 

 

 

In some cases, it is possible have a "Fix ..." action, in which Model Advisor would follow a pre-defined or programmed fix action to correct the model for you. Since font type and size have a very "objective" procedure to fix the model, we can implement a "Fix ..." action by using the sl_customization.m file.

5. The Check Simulink window screen color also does not pass. Following the same procedure, use the Fix window screen color button to correct the window screen color.

The model should now look like below. The fonts have been updated to Tahoma, size 16 and the window screen color is white.
